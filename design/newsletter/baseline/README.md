# design/newsletter/baseline/

## 폴더 목적
"정상" 기준으로 삼을 두 시점의 근거 자료를 분리 보관한다: 2026-07-20 실제
발행본(`july20/`)과 2026-07-22 롤백 조치 근거(`rollback/`). Stitch 디자인이
"무엇으로부터 출발하는지"를 명확히 하기 위한 폴더다.

## 입력 파일
- `july20/` — 2026-07-20 06:41 실제 발행본의 HTML/스크린샷/이메일 원본(확보되면)
- `rollback/` — 2026-07-22 롤백 작업의 근거(git 커밋 SHA, diff, postmortem 링크)

## 출력 파일
없음(참고 자료 보관).

## production 여부
아니오.

## 사용자 승인 필요 여부
`july20/`에 넣을 실물 자료는 사용자 제공이 필요하다. `rollback/`은 이미
daily-briefing 저장소의 실제 커밋(공개 정보)을 요약한 것이라 별도 승인 없이
채워도 되나, 내용 자체(어떤 컴포넌트를 KEEP/REMOVE로 볼지)는 여전히 사용자
승인 대상이다.

## 금지 작업
- 확보되지 않은 2026-07-20 실제 발행본 스크린샷/원본을 추정으로 재구성하는 것
- git 커밋 3c9124f 이외의 근거 없는 "정상 기준" 주장을 추가하는 것
